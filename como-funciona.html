⚙ Documentación técnica

Cómo se hizo el renderizado

Guía paso a paso de cómo Map Viewer dibuja millones de puntos geográficos en tiempo real sin que el navegador colapse.

4M+ puntos renderizados
1 llamada al GPU por frame
0 objetos del DOM por punto
El problema y la solución
1

El problema: Google Maps Markers no escalan

La forma tradicional de pintar puntos en Google Maps es creando un objeto google.maps.Marker por cada punto. Cada marker es un nodo del DOM con su propio listener de eventos, posición y ciclo de vida.

Enfoque 100 puntos 3,000 puntos 40,000+ puntos
google.maps.Marker ✓ Fluido ⚠ Lento ✗ Crashea
deck.gl ScatterplotLayer ✓ Fluido ✓ Fluido ✓ Fluido
💥
¿Por qué crashea a los ~3,000 markers? El navegador tiene que mantener 3,000 nodos del DOM activos, recalcular la posición de cada uno en cada pan/zoom, y re-renderizar el layout. A partir de ~1,500 objetos el rendimiento cae; a ~3,000 el tab colapsa.
2

La solución: WebGL con deck.gl

deck.gl es una librería open source (MIT) de Uber/vis.gl que renderiza datos geoespaciales directamente en la GPU usando WebGL. En lugar de crear objetos del DOM, sube todos los puntos a la tarjeta gráfica como un buffer de datos y los dibuja en una sola llamada.

JSON (4M puntos)
→
JavaScript Array
→
Float32Array (GPU buffer)
1 drawCall → 4,000,000 puntos en pantalla
🚀
¿Por qué es tan rápido? La GPU está diseñada para procesar millones de operaciones en paralelo. Dibujar 4M puntos para la GPU es casi tan barato como dibujar 100, porque los procesa todos al mismo tiempo en sus miles de núcleos.
Implementación
3

GoogleMapsOverlay: conectar deck.gl al mapa

deck.gl se integra a Google Maps mediante GoogleMapsOverlay, que crea una capa WebGL transparente encima del mapa y le pasa los eventos del mouse (hover, click) a deck.gl. El mapa base de Google sigue funcionando normalmente debajo.

// 1. Crear el mapa de Google normalmente
const map = new google.maps.Map(element, {
  center: { lat: 0, lng: 0 }, zoom: 2
});

// 2. Crear el overlay de deck.gl
const overlay = new deck.GoogleMapsOverlay({
  layers: [],          // se agregan las capas aquí
  pickingRadius: 6     // radio de detección hover en px
});

// 3. Adjuntar al mapa (UNA sola vez)
overlay.setMap(map);
💡
El overlay vive durante toda la sesión. Se crea una sola vez al cargar el mapa y permanece. Para actualizar los puntos se usa overlay.setProps(), no se destruye y re-crea.
4

ScatterplotLayer: la capa que dibuja los puntos

ScatterplotLayer es la capa de deck.gl que renderiza círculos en coordenadas geográficas. Cada punto del JSON se convierte en un círculo dibujado por la GPU.

const layer = new deck.ScatterplotLayer({
  id: 'pts',             // ID único para el diffing
  data: PTS,            // array de objetos {pos:[lng,lat], p:{...}}

  getPosition: d => d.pos,  // accessor de posición
  radiusUnits: 'pixels', // radio en px de pantalla (no metros)
  getRadius: 4,           // tamaño fijo sin importar el zoom

  getFillColor: [245, 158, 11, 200],  // RGBA constante
  stroked: false,        // sin borde → ~2× más rápido
  pickable: true,        // habilita hover/click
});

overlay.setProps({ layers: [layer] });

Cada vez que se llama a setProps, deck.gl compara la nueva capa con la anterior por su id y solo actualiza lo que cambió. Si el array data tiene la misma referencia en memoria, no re-sube los datos a la GPU — solo cambia el color o el radio en el shader.

5

Por qué cambiar color o tamaño no re-dibuja todo

Esta es la clave del rendimiento. deck.gl separa los datos de las propiedades visuales:

Geometría (posiciones)
→
GPU buffer — sube 1 vez
Visual (color, radio)
→
Uniform del shader — cambia gratis
// Cambiar color → solo actualiza el uniform en el shader (microsegundos)
overlay.setProps({ layers: [new ScatterplotLayer({
  id: 'pts',
  data: PTS,            // ← misma referencia → NO re-sube a GPU
  getFillColor: [255, 0, 0, 200],  // ← solo esto cambia
})] });

// Agregar puntos → nueva referencia → SÍ re-sube a GPU (pero 1 sola vez)
PTS = [...PTS, ...newPoints];  // nuevo array
overlay.setProps({ layers: [new ScatterplotLayer({
  id: 'pts',
  data: PTS,            // ← nueva referencia → re-upload GPU
})] });
6

El slider de puntos visibles

El slider controla cuántos puntos del array se mandan a la capa en cada re-render. Usa Array.slice(0, n) para obtener los primeros N puntos.

// Al mover el slider:
function onVisSlider(val) {
  visibleCount = parseInt(val);

  // Debounce: espera 180ms para no saturar la GPU mientras arrastras
  clearTimeout(visDebounce);
  visDebounce = setTimeout(buildLayer, 180);
}

function buildLayer() {
  // Si mostramos todos, usamos PTS directamente (sin allocar)
  // Si mostramos menos, slice() crea sub-array (re-upload a GPU)
  const data = visibleCount >= PTS.length
    ? PTS
    : PTS.slice(0, visibleCount);

  overlay.setProps({ layers: [new ScatterplotLayer({ id:'pts', data, ... })] });
}
⚠
¿Por qué el debounce? Al arrastrar el slider rápido podrían dispararse 30+ llamadas por segundo. El debounce de 180ms asegura que la GPU solo re-dibuja cuando el usuario deja de mover el slider, evitando bloquear el hilo principal.
7

Auto-detección del formato JSON

Map Viewer acepta múltiples formatos sin que el usuario configure nada. La función parsePoints() detecta automáticamente la estructura:

function parsePoints(raw) {
  // 1. GeoJSON FeatureCollection
  if (raw?.type === 'FeatureCollection') { /* ... */ }

  // 2. Buscar el array en contenedores comunes
  let arr = Array.isArray(raw) ? raw : null;
  if (!arr) {
    for (const key of ['data','points','items','rows',...]) {
      if (Array.isArray(raw?.[key])) { arr = raw[key]; break; }
    }
  }

  // 3. Detectar estructura de cada elemento
  const first = arr[0];
  if (Array.isArray(first)) { // → [[lng, lat], ...]
    return arr.map(p => ({ pos: [+p[0], +p[1]] }));
  }
  const latK = findKey(first, ['lat','latitude','Lat',...]);
  const lngK = findKey(first, ['lng','lon','longitude',...]);
  return arr.map(p => ({ pos: [+p[lngK], +p[latK]], p }));
}

Todos los puntos inválidos (coordenadas fuera de rango, NaN, nulos) se filtran automáticamente antes de subir al mapa.

8

Flujo completo de datos

Este es el recorrido de un archivo JSON desde que se sube hasta que aparece en el mapa:

① FileReader.readAsText()
→
② JSON.parse(text)
→
③ parsePoints()
④ PTS = [{pos,p}, ...]
→
⑤ ScatterplotLayer(data: PTS)
→
⑥ GPU buffer → pantalla
📦
Sobre el uso de memoria. 4 millones de puntos como objetos JavaScript ocupan ~320 MB en RAM. Cuando deck.gl los sube al GPU los convierte a Float32Array (~32 MB en VRAM). Por eso la primera carga tarda unos segundos — es el JSON.parse + la transferencia CPU→GPU.
